# Evolution Graph Visualization Design

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a dual-view visualization system for topic evolution graphs with time-series and network perspectives

**Architecture:** Static HTML deployment to GitHub Pages using D3.js for timeline view and vis-network for network view, with Python preprocessing for data transformation

**Tech Stack:** React, D3.js, vis-network, Python, Vite

---

## Overview

This document specifies a visualization system for the evolution graph data generated by the academic-trend-monitor pipeline. The system provides two complementary views:

1. **Timeline View (Primary)**: Shows how topics evolve across time periods
2. **Network View (Secondary)**: Shows cross-topic relationships within a single time period

### Data Context

- **Input**: `data/output/evolution_graphs/{domain}_graph.json`
- **Current Scale**: ~4,302 nodes (Layer 3+), ~1,066,541 edges (with confidence filtering)
- **Time Periods**: 2025-02, 2025-03, 2024-04, 2025-05
- **Color Coding**: By TopicMode (theory/method/problem/hybrid)

## Component Specifications

### TimelineView Component

**Purpose**: Display topic evolution across time periods with continued edges

**Props Interface**:
```typescript
interface TimelineViewProps {
  data: {
    periods: string[];
    nodes: TopicNode[];
    edges: EvolutionEdge[];
  };
  filters: {
    categories: string[];
    confidenceThreshold: number;
    showCrossTopicEdges: boolean;
  };
  colorScheme: Record<TopicMode, string>;
}
```

**Visual Design**:
- Horizontal layout with time periods as columns
- Each column shows topics as vertically-arranged cards/nodes
- Curved Bezier lines connect same-topic across consecutive periods
- Node color = TopicMode (theory=blue, method=green, problem=orange, hybrid=purple)
- Node size = based on paper_count using logarithmic scale: `size = 8 + 12 * log2(paper_count + 1) / log2(max_count + 1)`, with fallback to fixed size 10px if paper_count is 0 or unavailable

**Interactions**:
- Hover: Show TopicTooltip with name, category, mode, period
- Click: Highlight this topic's full lineage across all periods
- Scroll: Horizontal pan across time periods
- Zoom: Scale both time and node density

**Edge Filtering**:
- Always show `continued` edges (confidence=1.0)
- Optionally show `diffused_to_neighbor` edges where confidence >= threshold

**Implementation Notes**:
- Use D3.js with Canvas rendering for performance
- Pre-calculate node positions in Python to reduce frontend computation
- Use requestAnimationFrame for smooth transitions

### NetworkView Component

**Purpose**: Display cross-topic relationships within a single time period

**Props Interface**:
```typescript
interface NetworkViewProps {
  data: {
    period: string;
    nodes: TopicNode[];
    edges: EvolutionEdge[];
  };
  options: {
    physics: boolean;
    clustering: boolean;
    edgeWeightThreshold: number;
  };
  colorScheme: Record<TopicMode, string>;
}
```

**Visual Design**:
- Force-directed graph layout
- Nodes: Colored by mode, sized by paper_count
- Edges: Styled by relation_type (solid for category edges, dashed for others)
- Physics simulation for dynamic layout

**Interactions**:
- Drag: Reposition nodes
- Scroll: Zoom in/out
- Hover: Show connected edges and neighbor nodes
- Click: Focus on node and its immediate neighbors

**Performance**:
- Use vis-network's clustering for large graphs
- Limit to single period (reduces nodes to ~1000)
- Physics simulation behavior:
  - ≤300 nodes: Full physics enabled by default
  - 300-800 nodes: Physics enabled but with reduced iterations
  - >800 nodes: Physics disabled, use static hierarchical layout
- Apply confidence threshold before rendering

### ViewSwitcher Component

**Purpose**: Toggle between Timeline and Network views

**Design**:
- Tab-style switcher at top of container
- Timeline tab: Icon + "演化时间轴"
- Network tab: Icon + "关系网络"
- Remember last selected view in localStorage

### ConfidenceSlider Component

**Purpose**: Control edge visibility by confidence threshold

**Design**:
- Range slider: 0.6 to 1.0
- Step: 0.05
- Label: "置信度筛选: {value}"
- Default: 0.8 (show only strong connections)

### CategoryFilter Component

**Purpose**: Filter nodes by arXiv category

**Design**:
- Horizontal button group
- Options: "全部", "math.OA", "math.NT", etc.
- Multi-select with visual active state
- "全部" button to clear selection

### TopicTooltip Component

**Purpose**: Display topic details on hover

**Content**:
- Topic name (bold)
- Category (e.g., math.OA)
- Mode (with color indicator)
- Period
- Paper count (if available)

## Data Preprocessing

### Python Script: evolution_visualization_prep.py

**Input**: `{domain}_graph.json`
**Output**: `{domain}_visualization.json`

**Processing Steps**:

1. **Layer 3+ Filtering**:
   - Load `aligned_topics_hierarchy.json`
   - Keep only topics with hierarchy depth >= 3
   - Filter nodes and edges accordingly

2. **Edge Classification**:
   - Separate `continued` edges (temporal)
   - Separate `diffused_to_neighbor` edges (cross-topic)

3. **Data Aggregation**:
   - Group nodes by period and category
   - Pre-calculate layout positions for Timeline view
   - Build adjacency lists for Network view

4. **Output Format**:
```json
{
  "version": "1.0",
  "domain": "math",
  "generated_at": "ISO timestamp",
  "metadata": {
    "total_nodes": 4302,
    "total_edges": 1066541,
    "filtered_nodes": 3400,
    "periods": ["2025-02", "2025-03", "2025-04", "2025-05"],
    "categories": ["math.OA", "math.NT", ...]
  },
  "timeline": {
    "periods": [...],
    "nodes": [...],
    "continued_edges": [...],
    "cross_topic_edges": [...]
  },
  "networks": {
    "2025-02": { "nodes": [...], "edges": [...] },
    "2025-03": { "nodes": [...], "edges": [...] },
    ...
  }
}
```

**Field Handling for Missing/Optional Data**:
- `paper_count`: Use 0 if null/undefined/empty array
- `embedding`: Omit from output (not needed for visualization)
- `related_paths`: Not required, ignore if missing
- `mode`: Default to "hybrid" if missing or invalid
- `name`: Use topic_id as fallback if empty
- `category`: Skip node if missing (should not happen in valid data)

**Tablet (768px-1023px)**: Stack controls vertically, maintain full functionality
**Mobile (<768px)**:
- Timeline view: Horizontal scroll required (keep node size)
- Network view: Fullscreen with pinch-zoom enabled
- Filters: Collapse into dropdown menu
- Hide less critical metadata in tooltips

## Performance Considerations

### Timeline View
- **Node Limit**: Display all Layer 3+ nodes (~3000-4000)
- **Edge Strategy**: Only render `continued` edges by default (~4000)
- **Rendering**: Canvas-based D3.js for smooth interaction
- **Optimization**: Pre-calculate node positions in Python

### Network View
- **Node Limit**: Single period only (~1000 nodes max)
- **Edge Strategy**: Apply confidence threshold before rendering
- **Physics**: Disable for >500 nodes, use clustering

### Data Loading
- Lazy load visualization data on component mount
- Show loading skeleton during fetch
- Cache data in memory to prevent re-fetch

## Error Handling

1. **Data Missing**: Show "数据准备中..." with instructions to run pipeline
2. **Empty Filter Result**: Show "无匹配主题" with reset filters button
3. **Render Error**: Use React Error Boundary component at EvolutionGraphContainer level
   - Display: "可视化渲染出错" with stack trace (dev) or generic message (prod)
   - Provide "重试" button to re-render
   - Provide "返回默认视图" button to reset filters and view state
4. **Loading State**: Show skeleton placeholder with pulsing animation during data fetch
