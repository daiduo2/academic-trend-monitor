# 动态层级钻取功能设计

> 为 DomainDashboard 添加 Layer 3/4 动态钻取能力

## 目标

让领域热度分析仪表盘支持动态层级导航：
- 用户选择 Layer 2 后，默认显示该 Layer 2 下的所有 Layer 3 主题热度
- 点击某个 Layer 3 柱子，图表切换显示该 Layer 3 下的 Layer 4 子主题
- 点击 Layer 4（叶子节点），弹出详情弹窗
- 面包屑导航支持快速返回上级

## 架构设计

### 导航模式：混合模式

| 层级 | 导航方式 | 说明 |
|------|----------|------|
| Layer 1/2 | 下拉选单 | 保持现有控件 |
| Layer 3/4 | 图表点击钻取 | 点击柱子进入下级 |
| 面包屑 | 始终可见 | 点击面包屑返回对应层级 |

### 组件结构

```
DomainDashboard
├── Controls（下拉选单区）
│   ├── 月份选择
│   ├── Layer 1 学科选择
│   └── Layer 2 子类选择
├── BreadcrumbNav（面包屑导航）
│   ├── Layer 2（可点击返回）
│   ├── Layer 3（可点击返回）
│   └── Layer 4（当前，不可点击）
├── Chart（动态柱状图）
│   ├── 点击柱子 → 钻取到子主题
│   └── 点击叶子节点 → 打开详情弹窗
├── Stats（统计卡片）
├── TopicDetailModal（主题详情弹窗）
└── TopicList（主题列表）
```

### 数据流

1. 用户选择 Layer 1/2 → 加载该 Layer 2 下所有主题
2. 提取所有主题的 `hierarchy_path` 构建层级树
3. 计算每个中间节点的聚合数据（子主题论文数之和）
4. 初始显示 Layer 3 级别的统计（按 path[2] 分组）
5. 点击某个 Layer 3 → 图表切换显示其下的 Layer 4 统计
6. 点击 Layer 4 → 打开详情弹窗

### 弹窗内容

- 主题名称、完整层级路径
- 关键词标签云（前10个）
- 13个月论文数量趋势迷你图
- "在趋势追踪中查看"按钮

## 数据结构

### 层级路径示例

```json
{
  "hierarchy_path": ["cs.CV研究", "视觉识别与理解", "通用视觉识别", "异常检测"],
  "hierarchy_depth": 4,
  "latest_paper_count": 45,
  "keywords": ["anomaly", "detection", ...]
}
```

### 层级树构建

从所有主题的 `hierarchy_path` 构建树形结构，计算每个节点的：
- `name`: 节点名称
- `paper_count`: 直接论文数 + 所有子节点论文数
- `children`: 子节点数组
- `topic_ids`: 该节点下的所有主题ID

## 关键交互

- **面包屑**: 始终显示完整路径 `cs.CV > 视觉识别与理解 > 通用视觉识别`
- **返回**: 点击面包屑的任意上级节点，图表直接跳转到该层级
- **颜色**: 每个 Layer 3 分配固定颜色，其子主题使用同色系渐变
- **空状态**: 若某 Layer 3 下无 Layer 4，点击直接显示详情弹窗

## 依赖

- 现有数据结构 `aligned_topics_hierarchy.json`
- 现有组件 `HierarchyTree`（可选，用于展示树形结构）
- Recharts 图表库（已集成）
