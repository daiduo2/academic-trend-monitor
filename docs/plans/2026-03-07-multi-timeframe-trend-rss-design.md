# 多时间粒度趋势分析与 RSS 订阅系统设计

## 设计概述

在现有月度主题建模基础上，增加**周度趋势分析**和**日度 RSS 订阅**功能，实现三层时间粒度的学术热点追踪系统。

**核心思路:**
- 月度: BERTopic 主题建模 (本地运行，增量更新)
- 周度: 趋势分析 (GitHub Actions，自动触发)
- 日度: RSS 订阅 (GitHub Actions，自动触发)
- 统一数据: 精简格式文献索引，多粒度复用

## 打标签算法

### 策略选择

**选用: BERTopic Embedding + 余弦相似度 (多标签阈值)**

| 方案 | 优劣 | 选择 |
|------|------|------|
| Cross-Encoder | 精度高，速度慢 | ❌ |
| **BERTopic Embedding** | 复用模型，简单快速 | ✅ |
| Hybrid | 复杂度高 | ❌ |

### 算法流程

```
输入: 文献元信息 (标题 + 摘要)
  │
  ▼
BERTopic Embedding → 384d 向量
  │
  ▼
与主题向量库计算余弦相似度
  │
  ▼
筛选相似度 > 0.6 的主题
  │
  ▼
输出: 多标签 [topic_5, topic_12, ...] + 分数 [0.82, 0.71, ...]
```

## 数据存储设计

### 精简格式 (GitHub Pages 适配)

**文献索引 (recent.jsonl):**
```json
{"i":"2503.12345","t":"Paper Title","a":["Alice","Bob"],"c":"AI","p":"250307","g":[5,12,18]}
```

| 字段 | 原字段 | 说明 |
|------|--------|------|
| `i` | id | arXiv ID |
| `t` | title | 标题 |
| `a` | authors | 作者 (只保留前3个) |
| `c` | category | 分类代码 (映射表) |
| `p` | published | 发布日期 (YYMMDD) |
| `g` | tags | 主题标签 ID 数组 |

**主题库 (topics.json):**
```json
{
  "version": "2025-03",
  "topics": {
    "5": {"n": "大语言模型对齐", "k": ["RLHF", "alignment"], "l": 3, "p": "AI"},
    "12": {"n": "多模态学习", "k": ["multimodal", "vision"], "l": 3, "p": "AI"}
  },
  "categories": {"AI": "cs.AI", "CV": "cs.CV", "LG": "cs.LG"}
}
```

### 存储预估

| 指标 | 数值 |
|------|------|
| 月文献量 | 2-4万篇 |
| 单篇大小 | ~300B (精简后) |
| 单月数据 | ~12MB |
| recent.jsonl (7天) | ~3MB |
| 压缩率 | 85% (相比原始 2KB/篇) |

### 文件结构

```
data/
├── topics.json              # 主题库 (月度更新)
├── weekly/
│   ├── 2025-W09.json       # 周度趋势
│   └── 2025-W10.json
└── recent.jsonl            # 最近7天文献 (日度更新)
```

## 工作流程

### 日度工作流 (GitHub Actions)

```yaml
name: Daily Update
on:
  schedule:
    - cron: '0 2 * * *'  # 每天 UTC 02:00

jobs:
  update:
    steps:
      - 获取当日 arXiv 新增文献
      - 向量匹配打标签
      - 追加到 recent.jsonl
      - 保留最近7天数据
      - 提交到 GitHub Pages
```

### 周度工作流 (GitHub Actions)

```yaml
name: Weekly Trend
on:
  schedule:
    - cron: '0 3 * * 0'  # 每周日 UTC 03:00

jobs:
  analyze:
    steps:
      - 读取 recent.jsonl
      - 按标签聚合统计
      - 计算热度变化趋势
      - 生成 weekly/YYYY-WXX.json
      - 提交到 GitHub Pages
```

### 月度工作流 (本地)

```bash
# 用户在本地运行
make monthly-modeling

# 内部流程:
# 1. 拉取当月所有数据
# 2. BERTopic 增量建模
# 3. 生成 topics.json
# 4. 可选: 重新打标 recent.jsonl
# 5. 推送到 GitHub Pages
```

## 关键技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 打标签输入 | 标题 + 摘要 | 元信息完整，BERTopic 兼容 |
| 打标签输出 | 多标签 (阈值 > 0.6) | 覆盖跨学科文献 |
| 月度建模策略 | 增量更新 (不删主题) | 避免主题漂移，订阅稳定 |
| 数据存储 | 精简格式 + 按月分片 | GitHub Pages 适配，控制体积 |
| RSS 生成 | 纯前端 blob URL | 无需后端服务 |
| 用户偏好 | LocalStorage | 简单，无后端依赖 |

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 数据量过大 | 精简格式 (85% 压缩)，去除摘要，短字段名 |
| 主题漂移 | 月度建模增量更新，不删除旧主题 |
| 浏览器性能 | 只加载最近7天数据，按需加载历史 |
| GitHub Pages 限制 | 静态文件，无计算，符合使用条款 |
