# 学术热点趋势分析仪表盘 - 设计方案

日期: 2026-03-04

## 项目目标

构建一个可部署到 GitHub Pages 的学术热点趋势分析仪表盘，支持从时间切片和领域切片两个维度展示研究热点。

## 核心架构决策

### 1. 静态数据 + 纯前端部署

- 数据每月在本地生成一次，导出为静态 JSON
- GitHub Pages 只托管展示页面，无后端服务
- 利用 GitHub Actions 实现数据更新后自动部署

### 2. BERTopic + LLM 层次化

- **BERTopic**: 每月独立建模，提取扁平主题
- **LLM (DeepSeek)**: 负责跨月主题对齐、层次化构建、语义命名

### 3. 层次结构设计

```
Layer 1: Discipline (学科)          [固定] - cs, math, physics...
Layer 2: Category (arXiv 分类)      [固定] - cs.AI, cs.CV...
Layer 3: Coarse Topic (粗主题)      [BERTopic + LLM 粗分]
Layer 4+: Fine Topics (细主题)      [LLM 层次化细分]
```

### 4. 主路径树 + 关联标签

- 每个主题有一个主路径（primary_path），用于树形导航
- 支持关联路径（related_paths），用于展示跨学科联系
- 多父节点选择策略：选择语义上更具体的作为 primary_parent

## 数据流

```
Raw Data (12个月 JSONL)
    ↓
BERTopic 每月建模 → 扁平主题列表
    ↓
LLM 第一轮：按 arXiv Category 粗分
    ↓
LLM 第二轮：每个 Category 内层次化
    ↓
LLM 第三轮：跨月主题对齐
    ↓
生成 JSON: topics_tree.json + trend_stats.json
    ↓
复制到 frontend/public/data/
    ↓
构建并部署到 GitHub Pages
```

## 数据格式

### topics_tree.json

```json
{
  "version": "2026-02",
  "topics": {
    "topic_15": {
      "id": "topic_15",
      "name": "Visual SLAM",
      "keywords": ["slam", "visual", "camera"],
      "paper_count": 120,
      "period": "2025-02",
      "primary_path": ["cs", "cs.RO", "Robot Navigation", "SLAM", "Visual SLAM"],
      "parent_id": "topic_8",
      "related_paths": [["cs", "cs.CV", "3D Vision", "SLAM"]],
      "is_leaf": true
    }
  },
  "tree": {
    "id": "cs",
    "name": "Computer Science",
    "children": [...]
  }
}
```

### trend_stats.json

```json
{
  "trends": {
    "topic_15": {
      "history": [
        {"period": "2025-01", "paper_count": 100, "rank": 15},
        {"period": "2025-02", "paper_count": 120, "rank": 12}
      ],
      "growth_rate": 0.2,
      "trend": "rising"
    }
  }
}
```

## 前端设计

### 技术栈

- React 18 + Vite
- D3.js（趋势图、树图）
- Tailwind CSS

### 视图

1. **时间切片视图**: 展示主题热度趋势，支持学科筛选
2. **领域切片视图**: 树形下钻浏览，从 Layer 1 到 Layer 6
3. **主题详情页**: 展示论文列表、关键词、趋势分析

## LLM 调用策略

针对 ~155 个主题/月：

| 步骤 | 调用次数 | 说明 |
|------|---------|------|
| 第一轮粗分 | ~1 次 | 一次性分类 155 个主题到 ~20 个粗主题 |
| 第二轮细分 | ~20 次 | 每个粗主题内部建层 |
| 第三轮对齐 | ~50 次 | 跨月主题匹配 |
| **总计** | **~71 次/月** | 完全可接受 |
